separateFilesByExts

mixed separateFilesByExts(string root_dir , string dest_dir_path , array exts_cut)

Description

Aucun paramètre n'est optionnel pour cette méthode.

Différemment de separateFilesByMaxSize(), cette méthode effectue bien une récursivité d'arborescence,
il faut donc faire attention à ce qu'il n'y ait pas TROP de sous dossiers et trop de fichiers !
Pour se prévenir d'un temps d'exécution trop important, comme expliqué ici, la méthode déplace d'office les fichiers au lieu de les copiers.

Différement de separateFilesByMaxSize(), le deuxième paramètre ne doit pas forcément pointer vers un dossier déjà existant,
mais vous pouvez spécifier un nom de dossier conteneur de tous les autres, qui porteront le nom de chaque extension contenu dans
le troisième paramètre.

Encore une fois, la méthode renvoi vrai si tout s'est bien passé ou un tableau avec les fichiers en erreur
si des erreurs sont survenues.

Exemple




$o_recur=new RFunk;

$a_all_exts=$o_recur->getAllExts('C:\Documents and Settings\pouet\Mes documents');

$m_retour=$o_recur->separateFilesByExts('C:\Documents and Settings\pouet\Mes documents\', getcwd().RFunk::DS.'_root', $a_all_exts);

if(is_bool($m_retour))
{
	echo 'le rassemblement de fichiers semble s\'etre bien déroulé';
	
}elseif(is_array($m_retour))
{
	echo 'attention pour diverses raisons';

	foreach($m_retour as $i_key => $s_value)
	{
		echo '<br />ce fichier n\'a pu etre copié ou déplacer => '.$s_value;
	}
}
