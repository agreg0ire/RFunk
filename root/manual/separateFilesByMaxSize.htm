separateFilesByMaxSize

mixed separateFilesByMaxSize( string root_dir [, string dest_path [, string dest_dir_name [, integer limit_in_mo ]]])

Description

Cette méthode un peu spéciale n'a pas une "récursivité d'arborescence" mais est récursive dans le meme répertoire.
Il faut donc avoir stocké auparavant tous les éléments dans UN SEUL ET UNIQUE REPERTOIRE SOURCE que vous voulez séparés en tailles limite dans des dossiers pour etre zippés après par exemple.

Attention pour que la récusivité ne soit pas infinie, la méthode déplace chaque fichiers.

Le chemin de destination par défaut vaut '.', le nom de dossier dans lequels seront stockés tous les autres dossiers est par défaut "_root",
et la taille de séparation est fixée par défaut à 2MO

Attention, des comportements étranges peuvent survenir si vous fixer une taille limite inférieure à 2 mo.
Bien sur, votre dossier source (qui ne doit pas contenir de sous dossier) doit contenir un total de poids de fichier supérieur à celui indiqué dans votre limite de séparation.

La méthode renvoi VRAI si tout s'est bien passé, ou un tableau avec les fichiers qui n'ont pas pu être déplacé si c'est le cas.

Exemple



$m_retour=$o_rfunk->separateFilesByMaxSize('C:\Documents and Settings\arthur\Mes documents\Mes images',getcwd(),'dir_dest',4);


if(is_array($m_retour))
{
	echo 'Ce(s) fichier(s) n'ont pu être déplacer =>';
	
	foreach($m_retour as $i_key => $s_value):  echo '<br />'.$s_value; endforeach;

}elseif(is_bool($m_retour))
{
	echo 'la séparation des fichiers s'est bien déroulée.';

}
