writeXmlTree

NULL writeXmlTree( string root_dir , string dest_path [, string xml_name [, string encoding ]])

Description

Vous devez impérativement fermer vous-même l'XML déclaré comme global dans la méthode (donc accessible en dehors de la classe), si vous voulez qu'il soit "syntaxiquement" valide.
La méthode aurait pu se charger de le faire mais cela aurait doublé son temps d'exécution car en effet,
pour pouvoir déterminer à quel moment fermer le noeud racine de l'xml (firstChildNode), un appel a getDirsPath() aurait du être établi au premier appel récursif de writeXmlTree(), pour compter le nombre de sous répertoire renvoyé par le tableau de getDrisPath().

Les deux derniers paramètres sont optionnels, ils correspondent successivement et respectivement aux :

- nom que portera votre arbre xml et
- l'encodage qui est par défaut en latin "iso-8859-1".

Exemple



$o_rfunk = new RFunk(180);

$o_rfunk->writeXmlTree(getcwd(),getcwd(),'retour_tree','UTF-8');

####" Fermeture manuel de l'XML ####

fwrite($h_file,' </Tree>'.RN);
fclose($h_file);
